SocksProxy.java - точка входа:
- принимает порт для запуска
- создает экземпляр прокси сервера
- запускает цикл оббработки соединений
________________________________________________
ProxyServer.java - основной сервер:
- инциализирует серверный сокет на указанном порту
- создает селектор для мониторинга событий от всех каналов
- создает dns-резолвер
- в цикле ждет событий от всех зарегестрированных каналов, обрабатывает
новые подключения клиентов, dns-ответы и делегирует обработку данных
клиентским сессиям

Селектор обеспечивает механизм мониторинга одного или нескольких каналов
NIO и распознавания того, когда один или несколько каналов становятся
доступными для передачи данных. Таким образом, один поток может
использоваться для управления несколькими каналами и, следовательно,
несколькими сетевыми подключениями.
________________________________________________
ClientSession.java - сессия клиента:
- управляет сессией клиента через состояние: начальное рукопожатие сокс5,
получение запроса на подключение, передача данных между клиентом и сервером,
сессия закрыта
- основные методы:
    - handle - обработка событий (чтение/запись/подключение),
    - handleClientRead - читает данные от клиента,
    - handleRemoteRead - читает данные от целевого сервера,
    - onResolved - колбэк от DNS резолвера,
    - connectDirect - устанавливает соединение с целевым сервером
________________________________________________
DnsResolver.java - резолвер:
- отправляет dns запросы через udp
- хранит ожидающие запросы с их колбэками
- при получении ответа ищет соответствующий колбэк и вызывает его
- dnsjava для формирования и парсинга dns - пакетоа
________________________________________________
SocksRequest.java - парсер запросов:
- parse() - парсит байты в структуру socks запроса
- +методы для отправки ответов клиенту
